# PharmaCare - Setup & Run Guide

This document explains the commands needed to start the PharmaCare application. You have two options to run the application: using Docker (Recommended) or running it manually in separate terminals.

---

## Option 1: Run with Docker (Recommended & Easiest)
This is the easiest method. It will automatically build the backend, the frontend, and start a dedicated MySQL database on port `3307` so it doesn't conflict with any local MySQL servers you might have.

1. Open a terminal in the root project folder (`/pharma`).
2. Run the following command:
   ```powershell
   docker-compose up -d --build
   ```
3. Wait a few minutes for the build to finish. 
4. The application is now running! 
   - **Frontend UI:** Access it at `http://localhost:5173`
   - **Backend API:** Available at `http://localhost:8081/api`

**To stop the Docker containers:**
```powershell
docker-compose down
```

---

## Option 2: Run Manually (For Development)

If you are actively coding and want to see the terminal logs for debugging, you can start the frontend and backend manually.

### 1. Database Setup
Ensure you have a local MySQL server running on port `3306` with the following credentials:
- **Username:** `root`
- **Password:** `Manohar@7485`
*(The backend will automatically create the `pharma_db` database for you).*

### 2. Start the Backend (Spring Boot)
Open a new terminal and navigate to the `backend` folder:
```powershell
cd backend
```
Start the backend by running this exact command:
```powershell
C:\Users\abbur\Downloads\apache-maven-3.9.15-bin\apache-maven-3.9.15\bin\mvn.cmd spring-boot:run
```
*(The backend server will start on port `8081`)*

### 3. Start the Frontend (Vite/React)
Open a second terminal and navigate to the `frontend` folder:
```powershell
cd frontend
```
Install the node dependencies (if you haven't already):
```powershell
npm install
```
Start the development server:
```powershell
npm run dev
```
*(The frontend will start on port `5173` and will automatically connect to your backend).*

## Medicine Images — Sharing the Project

Medicine images uploaded by admin are stored inside the project at:
  backend/src/main/resources/static/uploads/medicines/

This folder is included in the zip so images work on any machine without re-uploading.

If images are still not showing after unzipping:
1. Make sure the uploads folder exists at:
   pharma/pharma/backend/src/main/resources/static/uploads/medicines/

2. If you have images in the old location (backend/uploads/), copy them:
   Windows:
     xcopy /E /I "backend\uploads" "backend\src\main\resources\static\uploads"

3. Restart the backend:
     cd pharma/pharma/backend
     mvn spring-boot:run

Images are served at: http://localhost:8081/uploads/medicines/<filename>
